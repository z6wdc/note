gitmessage guide

.gitmessage 是一個純文字檔案，用來定義 git commit 訊息的預設格式。

透過這個機制，可以提示開發流程中的 commit 格式，提升可讀性與一致性。

如何設定 commit template

  1. 建立 commit template 檔案
  1. 編輯內容（範例見下方）
  1. 設定 git 使用該檔案
  1. 確認設定

.gitmessage 範例格式

常見的 Commit 類型（Conventional Commits）

類型 說明
feat 新功能
fix 錯誤修正
docs 文件變更
style 格式（不影響程式邏輯）
refactor 重構（非功能變更）
test 增加測試
chore 其他更新（建置、工具等）

使用建議與好處

  • 確保團隊成員使用一致的 commit message
  • 提升專案可維護性與回溯紀錄
